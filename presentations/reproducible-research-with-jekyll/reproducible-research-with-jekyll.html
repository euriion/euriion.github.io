재현 가능한 연구란?

재현 가능한 연구의 정의

재현 가능한 연구 (Reproducible Research)

  • 동일한 결과: 같은 데이터와 방법으로 같은 결과 도출
  • 투명성: 연구 과정과 방법론 공개
  • 코드 공유: 분석 코드와 스크립트 제공
  • 데이터 접근성: 원본 데이터 또는 샘플 데이터 제공

flowchart TD
    A[원본 데이터] --> B[분석 코드]
    B --> C[결과]
    C --> D[보고서/논문]
    D --> E[재현 검증]
    E --> F[신뢰성 확보]

재현 가능한 연구의 중요성

  • 과학적 신뢰성: 연구 결과의 검증 가능성
  • 협업 효율성: 팀원 간 연구 과정 공유
  • 지식 전수: 후속 연구자들의 학습 자료
  • 오류 발견: 동료 검토를 통한 품질 향상
  • 연구 확장: 기존 연구의 발전과 확장

연구 재현성의 위기

최근 연구에 따르면, 많은 과학 연구들이 재현되지 않는 문제가 대두되고 있습니다. 특히 심리학, 의학, 경제학 분야에서 이러한 문제가 심각합니다.

Jekyll 소개

Jekyll이란?

Jekyll은 Ruby 기반의 정적 사이트 생성기

  • Ruby 기반: 강력한 템플릿 엔진
  • Markdown 지원: 쉬운 콘텐츠 작성
  • GitHub Pages: 무료 호스팅 지원
  • 플러그인: 풍부한 확장 기능
  • 테마: 다양한 디자인 옵션

Jekyll의 장점

  • 연구자 관점
  • 기술적 관점
  • 사용자 관점
  • 버전 관리: Git을 통한 연구 과정 추적
  • 협업: 여러 연구자가 동시에 작업 가능
  • 재현성: 코드와 결과를 함께 관리
  • 문서화: 자동화된 문서 생성
  • 빠른 속도: 정적 사이트의 빠른 로딩
  • 보안: 데이터베이스 없는 안전한 구조
  • 확장성: 플러그인을 통한 기능 확장
  • 무료: GitHub Pages 무료 호스팅
  • 접근성: 웹 브라우저로 쉽게 접근
  • 검색: 구글 등 검색 엔진 최적화
  • 반응형: 모바일 친화적 디자인
  • 공유: URL을 통한 쉬운 공유

GitHub Pages 배포

GitHub Pages 설정

1. 저장소 생성 - GitHub에서 새 저장소 생성 - 저장소명: username.github.io 또는 프로젝트명 - Public으로 설정

2. 로컬 연결

git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/username/repo.git
git push -u origin main

3. Pages 활성화 - Settings → Pages - Source: Deploy from a branch - Branch: main / (root) - Save

4. 자동 배포 - 코드 변경 시 자동으로 사이트 업데이트 - 보통 1-2분 내에 반영 - https://username.github.io/repo 에서 확인

GitHub Actions 워크플로우

.github/workflows/jekyll.yml

name: Build and Deploy Jekyll Site

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    
    steps:
    - name: Checkout
      uses: actions/checkout@v2
      
    - name: Setup Ruby
      uses: ruby/setup-ruby@v1
      with:
        ruby-version: 3.0
        bundler-cache: true
        
    - name: Build Jekyll site
      run: bundle exec jekyll build
      
    - name: Deploy to GitHub Pages
      uses: peaceiris/actions-gh-pages@v3
      with:
        github_token: ${{ secrets.GITHUB_TOKEN }}
        publish_dir: ./_site

Best Practices

연구 문서화 원칙

  • 명확한 문서화: 모든 분석 과정을 상세히 기록
  • 버전 관리: 코드와 데이터의 변경 사항 추적
  • 협업 고려: 다른 연구자가 이해하기 쉽게 작성
  • 정기적 업데이트: 연구 진행에 따라 지속적으로 업데이트
  • 데이터 보안: 민감한 데이터는 별도 관리
